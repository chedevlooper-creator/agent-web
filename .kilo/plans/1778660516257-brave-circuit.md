# Hermes Benzeri Agent Sistemi — Geliştirme Planı

**Hedef:** Mevcut agent-web projesini, Hermes Agent benzeri kapalı öğrenme döngüsüne sahip, skill üretebilen, MCP entegre, subagent destekli bir otonom agent sistemine dönüştürmek.

**Kararlar:**
- Auth: Single-user local (multi-user yok)
- MCP Transport: SSE (sunucu tarafı events)
- Deployment: Docker production + local dev

---

## Mevcut Durum

- **Frontend:** Next.js 16, React 19, Tailwind 4, shadcn/ui
- **AI:** Vercel AI SDK 4, @ai-sdk/openai (OpenAI/OpenRouter/OpenCode provider)
- **State:** Zustand (client-side), trpc kurulu ama kullanılmıyor
- **Database:** Drizzle ORM + SQLite (@libsql/client)
- **Workspace:** `apps/web`, `packages/core`, `packages/db`

**Mevcut Schema (güncellenmiş):** `sessions`, `messages`, `memory_entries`, `skills`, `mcp_servers`, `subagents`, `tool_executions`

## Phase 1: Temel Altyapı — Veritabanı & API Katmanı

### 1.1 Database Setup
- `packages/db/src/schema.ts` zaten genişletilmiş.
- `packages/db/src/index.ts` export'ları güncelle (yeni tablo tipleri).
- `packages/db/package.json` içinde `db:push` script'i ile schema SQLite'a uygulanacak.

**Dosyalar:**
- `packages/db/src/index.ts` — yeni export'lar ekle
- `packages/db/drizzle.config.ts` — mevcut, dokunma

### 1.2 Core Paketi — Gerçek Tool Implementasyonları
Tool'lar şu anda API route'unda mock. `packages/core` içindeki tool'lar gerçekten çalışmalı.

**Dosyalar:**
- `packages/core/src/tools/terminal.ts` — `child_process.spawn` ile shell execution, timeout, output limit, path sanitization
- `packages/core/src/tools/file.ts` — `fs/promises` ile read/write/list/search, path traversal koruması
- `packages/core/src/tools/web.ts` — DuckDuckGo HTML scrape (search) + fetch + text extraction (scrape)
- `packages/core/src/tools/registry.ts` — skill'leri runtime'da DB'den çekip register etme metodu ekle
- `packages/core/src/index.ts` — tüm yeni modülleri export et

### 1.3 Chat API Route — Tam Entegrasyon
Mevcut `/api/chat` sadece stream ediyor. Tool'ları gerçekten execute etmeli, memory'yi okumalı, session'a kaydetmeli.

**Yeni yapı:**
- `apps/web/app/api/chat/route.ts` — refaktör
  - Request'ten `sessionId` al
  - Session geçmişini DB'den oku ve messages'a ekle
  - Memory manager'dan ilgili memory entry'lerini al ve system prompt'a inject et
  - `streamText` çağrısında tool'ları `packages/core` registry'sinden al
  - Tool çağrıları gerçekten execute et, `toolExecutions` tablosuna logla
  - Assistant mesajını DB'ye kaydet
  - Stream formatını client'a düzgün aktar

**Ara katman:**
- `packages/core/src/chat/engine.ts` — chat orchestration logic (memory inject, tool exec, skill inject)

## Phase 3: Bellek Sistemi (Memory)

### 3.1 Core — Memory Manager
- `packages/core/src/memory/manager.ts`
  - `addEntry(key, value, category, importance, sessionId?)` — kaydet
  - `getRelevant(query, limit?)` — keyword search + recency sıralaması (semantic search yok, SQLite FTS5 opsiyonel)
  - `getByCategory(category)` — filtrele
  - `summarizeSession(sessionId)` — session mesajlarını özetleyip memory'ye kaydet

### 3.2 API Routes
- `apps/web/app/api/memory/route.ts` — GET (list/query), POST (create)
- `apps/web/app/api/memory/[id]/route.ts` — DELETE

### 3.3 Chat Engine Entegrasyonu
Her chat isteğinde:
1. Memory manager'dan kullanıcı tercihleri ve geçmiş önemli bilgileri al
2. System prompt'un sonuna inject et (örnek: "User prefers TypeScript. User is working on a Next.js project...")
3. Yeni önemli bilgiler tespit edilirse otomatik `memory_entries` kaydet

### 3.4 Frontend — Memory Browser
- `apps/web/components/memory-panel.tsx` — memory entry'leri görüntüle, ekle, sil, kategoriye göre filtrele

- `packages/core/src/skills/manager.ts`

## Teknik Notlar & Kararlar

### Güvenlik (Single-User Local)
- Terminal tool'u: çalışma dizini dışına çıkmaz, timeout var, output limit var, bazı komutlar blocklist'te
- File tool'ları: `path.resolve` + `startsWith(READ_BASE)` traversal koruması
- Skill execution: `new Function()` ile çalıştırma, skill kodu sadece mevcut tool handler'larını çağırabilir. İleri seviyede VM2/QuickJS düşünülebilir.

### Memory Search
- Semantic search (vektör/embedding) bu versiyonda **yok**. Basit keyword search + recency + importance sıralaması kullanılacak.
- İleride `sqlite-vec` veya benzeri eklenebilir.

### State Yönetimi
- **Client:** Zustand (chat UI state, settings)
- **Server:** Drizzle ORM (kalıcı veri)
- **tRPC:** Kurulu ama kullanılmıyor. Plan kapsamında REST API (Next.js Route Handlers) ile devam edilecek. tRPC sonraki refactor'da değerlendirilebilir.

### LLM Provider'lar
- Mevcut: OpenAI, OpenRouter, OpenCode
- Yeni eklenmeyecek. Subagent'lar ana session'dan farklı model/provider kullanabilir.

## Riskler & Mitigasyonlar

| Risk | Mitigasyon |
|------|-----------|
| MCP SSE implementasyonu karmaşık | Basit event-source + JSON-RPC el ile yazılacak, `@modelcontextprotocol/sdk` sonradan eklenebilir |
| Skill execution güvenliği | `new Function()` ile başlanacak, sadece whitelisted API'lere erişim |
| Tool execution uzun sürerse stream kopabilir | AI SDK `maxSteps` + timeout yönetimi, uzun işlemler için async subagent |
| SQLite concurrency (Next.js dev HMR) | `file:./local.db` → absolute path, prod'da volume mount |
| Web scraping kırılabilir | DuckDuckGo HTML yapısı değişirse, fallback: kullanıcıya "search API key girin" denilecek |
