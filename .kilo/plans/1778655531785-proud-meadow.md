# agent-web Geliştirme Planı

**Oluşturulma:** 2026-05-13
**Hedef:** agent-web'i, Docker içinde geliştirilebilen, Hermes Agent'dan ilham alan, otonom yeteneklere sahip bir AI ajan platformuna dönüştürmek.

---

## Mevcut Durum Özeti

- **Stack:** Next.js 16.2.6, React 19, TypeScript 5, Tailwind CSS v4, shadcn/ui
- **Monorepo:** Turborepo + pnpm workspace (`apps/web`, `packages/core`, `packages/db`)
- **Docker:** Geliştirme container'ı hazır (`docker-compose up` ile çalıştırılabilir)
- **DB:** Drizzle ORM + libsql (SQLite) — şema tanımlı ama kullanılmıyor
- **AI:** Vercel AI SDK v4 (`streamText`, `@ai-sdk/openai`)
- **State:** Zustand (client-side only, `localStorage`'a persist ediyor)
- **Kritik Sorun:** `packages/core` ve `packages/db` atıl durumda; `apps/web` inline kod kullanıyor

---

## Strateji: 3 Fazlı Yol Haritası

### Faz 1: Temel Altyapı (Foundation) — 2-3 hafta
**Amaç:** Mevcut atıl paketleri aktifleştirmek, veritabanını entegre etmek, API güvenliğini sağlamak.

#### 1.1 `packages/core`'u Aktifleştir
- `apps/web/app/api/chat/route.ts`'teki inline provider/tool kodunu kaldır.
- API route, `packages/core`'dan `chatStream`, `registry`, `LLMConfig`, `Message` tiplerini import etsin.
- `packages/core/src/llm/client.ts`'teki `convertToModelMessages` hatası zaten düzeltildi; `tool_call` parsing doğru çalıştırılmalı.

#### 1.2 `packages/db`'yi Aktifleştir
- `apps/web` içinde `@agent-web/db`'yi kullanmaya başla.
- `DATABASE_URL` environment variable'ını Docker compose ve `.env`'de ayarla.
- `drizzle-kit push` ile schema'yı SQLite'e uygula.

#### 1.3 Veritabanı Şeması Genişletme
Mevcut `sessions` + `messages` tablolarına ek olarak:

```sql
-- Bellek sistemi
memory_entries (id, target, content, created_at, updated_at)
  target: 'memory' | 'user'

-- Skill sistemi
skills (id, name, description, content, category, created_at, updated_at)

-- Oturum özetleri (context compression)
session_summaries (id, session_id, summary, compressed_turn_count, created_at)
```

#### 1.4 API Key Yönetimi
- Client-side `localStorage`'daki `apiKey`'yi kaldır.
- Server-side `.env` dosyasında `OPENROUTER_API_KEY`, `OPENAI_API_KEY` gibi key'leri tut.
- Settings panel'de artık key **girilmeyecek**; yalnızca provider/model seçilecek. Key'ler server'dan gelecek.

#### 1.5 Tool Handler'larını Gerçekleştir
`packages/core/src/tools/*.ts`'teki placeholder handler'ları çalışır hale getir:
- `file.ts`: `read_file`, `write_file`, `search_files` — `safePath` ile path traversal korumalı.
- `terminal.ts`: `terminal` — Docker container içinde `child_process` ile çalıştır.
- `web.ts`: `web_search` (DuckDuckGo API veya OpenRouter web search), `web_scrape` (`fetch` + HTML parsing).

#### 1.6 Chat UI Düzenlemeleri
- `chat-interface.tsx`'teki manuel stream parsing'i Vercel AI SDK `useChat` hook'una geçir. Bu, kodu %60 azaltır ve tool call parsing'i güvenilir hale getirir.
- `react-syntax-highlighter/dist/cjs/styles/prism` import'unu ESM yoluna (`esm`) çevir.
- Mesajları artık `localStorage`'da değil, server'dan (SQLite DB) çek.

#### 1.7 Session CRUD API
- `GET /api/sessions` — oturum listesi
- `POST /api/sessions` — yeni oturum
- `GET /api/sessions/[id]/messages` — oturum mesajları
- `DELETE /api/sessions/[id]` — oturum sil

### Faz 2: Hermes-Inspired Özellikler — 3-4 hafta
**Amaç:** Agent'ın kendini geliştirmesini, hatırlamasını ve skill oluşturmasını sağlayan çekirdek sistemleri inşa etmek.

#### 2.1 Persistent Memory (MEMORY.md / USER.md Konsepti)
- Her kullanıcı için iki sınırlı metin bloğu tut (Hermes'teki gibi):
  - `MEMORY` (~2000 karakter): Ortam, proje, konvansiyonlar
  - `USER` (~1300 karakter): Kullanıcı profili, tercihler
- Her oturum başında bu bloğlar system prompt'a enjekte edilir.
- Agent'a `memory` tool'u ver: `add`, `replace`, `remove` action'ları.
- Kapasite dolduğunda agent'ı konsolidasyona zorla (LLM'e "yer açmak için eski kayıtları birleştir" deneği).

#### 2.2 Context Compression
- Konuşma uzadıkça (token limitinin %50'si) eski mesajları özetle.
- `session_summaries` tablosunda tut.
- Özetleme için ucuz/fast bir model kullan (Gemini Flash veya gpt-4o-mini).

#### 2.3 Session Search (FTS5)
- `messages` tablosuna FTS5 full-text search indeksi ekle.
- Agent'a `session_search` tool'u ver; eski konuşmalarda arama yapabilsin.
- Sonuçları Gemini Flash ile özetle.

#### 2.4 Skills Sistemi
- **SKILL.md formatı:** YAML frontmatter + markdown body
  ```markdown
  ---
  name: deploy-vercel
  description: Vercel deploy adımları
  ---
  # Deploy Procedure
  1. Run `pnpm build`
  2. Run `vercel --prod`
  ```
- Skills `~/.hermes/skills/` benzeri bir dizinde (`data/skills/`) veya DB'de tutulur.
- Progressive disclosure: `skills_list()` → `skill_view(name)`
- Agent-managed skills: Karmaşık görev sonrası `skill_manage` tool'u ile skill oluşturur.

#### 2.5 Approval Flow (Güvenlik)
- Tehlikeli komut listesi: `rm -rf`, `mkfs`, `dd`, `curl | sh`, vb.
- UI'da inline approval modal: `[Once] [Session] [Always] [Deny]`
- Hardline blocklist: `rm -rf /`, fork bomb, vb. — **hiçbir koşulda** çalışmaz.
- YOLO mode (`/yolo`): Kullanıcı bilerek onayları bypass edebilir (dev ortamı için).

#### 2.6 SSRF Koruması
- `web_search`, `web_scrape`, `browser_navigate` tool'larında URL validasyonu.
- RFC 1918, loopback, link-local, cloud metadata adresleri engellenir.

### Faz 3: Gelişmiş Özellikler — 4+ hafta
**Amaç:** Terminal sandbox, MCP entegrasyonu, ve gateway yetenekleri.

#### 3.1 Terminal Sandbox (Docker-in-Docker)
- Next.js API routes'da `child_process` ile shell komutu çalıştırma implemente et.
- **Güvenlik:** `child_process` doğrudan kullanmak yerine, Docker socket'i mount edip ayrı bir container başlatan bir wrapper yaz.
- Container hardening: `--cap-drop ALL`, `--security-opt no-new-privileges`, `--read-only`.
- Background process yönetimi: `terminal(background=true)` ile başlat, `process` tool'u ile yönet.

#### 3.2 MCP (Model Context Protocol) Client
- `@modelcontextprotocol/sdk` ile MCP client implementasyonu.
- HTTP/SSE transport ile remote MCP server'lara bağlan.
- Tool isimlerini `mcp_<server>_<tool>` prefix'i ile registry'e kaydet.
- Per-server `include/exclude` filtering UI'ı.

#### 3.3 Browser Automation
- Server-side `playwright` veya `puppeteer` ile browser otomasyonu.
- `browser_navigate`, `browser_snapshot`, `browser_vision`, `browser_click` tool'ları.

#### 3.4 Gateway / Multi-Platform (Opsiyonel)
- Telegram Bot API webhook
- Discord webhook
- Slack event subscription
- Bu, uzun süreli WebSocket bağlantıları gerektirir; serverless (Vercel) uygun değil. Docker container içinde çalışır.

#### 3.5 Cron / Zamanlanmış Görevler
- `node-cron` ile zamanlanmış job'lar.
- Job'ları SQLite'da tut (`cron_jobs` tablosu).
- Agent'ın belirli saatlerde otomatik çalışması.

## Teknik Kısıtlar ve Kararlar

| Karar | Açıklama |
|-------|---------|
| **Docker Geliştirme** | Tüm geliştirme Docker container'ında yapılır. `docker-compose up` ile başlar. |
| **Self-Hosted** | Vercel yerine Docker'da çalışır. Terminal sandbox, MCP stdio, persistent process için gerekli. |
| **SQLite** | Başlangıçta SQLite (libsql) yeterli. İleride PostgreSQL'e geçiş planlanabilir. |
| **TypeScript** | Tüm backend (tool'lar, memory, skills) TypeScript ile yazılır. Python (Hermes) yerine TS tercih edilir çünkü proje zaten TS. |
| **Next.js App Router** | API routes (`app/api/*`) kullanılır. Pages Router'a geçiş yok. |

## Başlangıç Komutları

```bash
# Container'ı başlat (host port 3001'e yönlendirilmiş)
docker compose up --build -d

# Container'a gir
docker compose exec app bash

# DB schema push (container içinde)
pnpm --filter @agent-web/db db:push

# Package build (container içinde)
pnpm --filter @agent-web/core build
pnpm --filter @agent-web/db build

# Logları izle
docker compose logs -f app
```

## Riskler ve Önlemler

| Risk | Önlem |
|------|-------|
| Terminal sandbox güvenliği | Docker container isolation, hardline blocklist, approval flow |
| API key sızıntısı | Server-side `.env`, client'a hiçbir key gitmez |
| Path traversal | `safePath` fonksiyonu, çalışma dizini sandbox'ı |
| Prompt injection | Context file (AGENTS.md) taraması, invisible Unicode kontrolü |
| Token limit aşımı | Context compression, iteration budget pressure |
